Feedback & Suggestions

Introduction

Here you can freely share your personal experiences, provide feedback about the First Cloud Journey (FCJ) program to help the team improve any shortcomings. Your feedback is invaluable!

General Program Assessment

1. Course Content

Strengths:

  • AWS knowledge is presented from basic to advanced levels, suitable for beginners
  • The 13-week structure is clear and helps learners grasp knowledge systematically
  • Topics are logically arranged, from basic services to complex architectures
  • Good balance between theory and practical exercises

Areas for Improvement:

  • Add more real-world examples from companies using AWS
  • Update faster when AWS launches new features
  • Add more interactive labs (sandbox) for hands-on practice
  • Provide detailed case studies from enterprises

2. Mentor Support & Guidance

Positive Aspects:

  • Mentors are always ready to help and answer questions in detail
  • Explanations are clear when I don’t understand an issue
  • Encourages learners to ask questions and find independent solutions
  • Feedback is provided in a timely manner, enabling quick improvement

Areas for Enhancement:

  • Provide more 1-on-1 sessions outside working hours for struggling learners
  • Establish fixed office hours so learners can schedule appointments
  • Create an FAQ document from frequently asked questions
  • Record Q&A sessions so learners can review them later

3. Community & Peer Interaction

Positive Points:

  • Friendly learning atmosphere, everyone is willing to help each other
  • Group study sessions are very helpful
  • Active chat groups where people share experiences and resources

Suggestions to Strengthen:

  • Organize more project showcase sessions from other learners
  • Create pair programming sessions between peers
  • Build an alumni community to maintain connections after the program
  • Create interest-based groups (Serverless, DevOps, Data Engineering, etc.)

4. Resources & Facilities

Positive Aspects:

  • Free AWS credits provided for hands-on practice
  • Comprehensive learning materials (AWS docs, blogs, videos)
  • Easy-to-use and stable learning platform

Improvement Proposals:

  • Provide a shared GitHub repository with all code examples
  • Create a cheat sheet or quick reference guide for common AWS services
  • Provide templates for commonly-used projects
  • Organize small competitions (hackathons) between learners with rewards

5. Course Pace & Balance

Good Aspects:

  • Reasonable learning pace, not overwhelming
  • Ability to learn at your own speed
  • Flexibility in assignment submission timelines

Possible Developments:

  • Add optional modules for those wanting deeper knowledge
  • Provide customized learning paths based on individual goals
  • Create nanodegrees or micro-credentials for milestone completions
